# Simple Volume Control 要件定義書

## 1. 概要

既存の Chrome 拡張 **Volume Master** は便利だが、UI がごちゃごちゃしており、見た目・操作感の面で好みに合わないため、  
**「音量制御機能に特化し、UI を極力シンプルにした自作 Chrome 拡張機能」** を開発する。

本拡張は、単なるページ内 `<audio>` / `<video>` 要素の操作ではなく、  
**アクティブタブの音声そのものを対象にした音量制御** を行う方針とする。

これにより、サイト実装への依存を減らし、より安定した挙動と完成度を目指す。

---

## 2. 開発方針

- **完成度重視**
- **UI は最小限**
- **機能は音量制御に絞る**
- 技術構成は **B案（tabCapture + offscreen document）** を採用する

---

## 3. 基本仕様

| 項目 | 内容 |
|---|---|
| 拡張機能名 | Simple Volume Control |
| 形式 | Chrome拡張機能（Manifest V3） |
| 対象ブラウザ | Google Chrome |
| 操作対象 | アクティブタブの音声 |
| 音量範囲 | 0% ～ 600% |
| デフォルト音量 | 100% |
| テーマ | ダークテーマ固定 |
| UI方針 | 小さく、ミニマル、視認性重視 |

---

## 4. この拡張で解決したいこと

### 4-1. 解決したい課題
- Volume Master の UI がごちゃごちゃしていて使いづらい
- シンプルに、今開いているタブの音量だけ調整したい
- YouTube などでは動画ごとに音量感が異なるため、手軽にブースト / 減衰したい

---

## 5. 機能要件

### 5-1. 音量スライダー
- 0% ～ 600% の範囲で音量を調整できる
- スライダー操作に応じて、リアルタイムで音量が変化する
- 現在値を `120%` のように数値表示する
- 小数は基本的に扱わず、表示は整数パーセントとする

### 5-2. リセットボタン
- クリックすると音量を **100%** に戻す
- スライダー表示も 100% に戻る

### 5-3. 現在状態の表示
- popup を開いたとき、現在のタブに対する音量を表示する
  - 現在音量（例: `150%`）

---

## 6. 非機能要件

### 6-1. 使いやすさ
- 1画面で完結する
- 余計な機能を入れない
- 初見でも操作方法が分かる

### 6-2. 視認性
- ダークテーマ固定
- コントラストを明確にする
- 数値が見やすいことを優先する

### 6-3. パフォーマンス
- popup の表示は軽量であること
- 音量変更時の遅延を極力少なくする
- 通常利用でブラウザ操作を阻害しないこと

### 6-4. 保守性
- UI層と音声制御層を分離する
- popup 側に重い処理を持たせない

## 8. 技術設計

### 8-1. 採用アーキテクチャ

**tabCapture + service worker + offscreen document + popup UI**

### 8-2. 採用理由

* popup は閉じると処理継続に向かないため、UI専用に分離したい
* 音量制御の本体は popup 外で動かしたい
* タブ音声全体を扱うことで、サイト依存を減らしたい
* 単純な content script より完成度を上げたい

### 8-3. 構成図

```text
popup (UI)
  ↓ メッセージ送信
service_worker
  ↓ stream ID 発行 / 状態管理
offscreen document
  ↓
Web Audio API
  - MediaStreamSource
  - GainNode
  - DynamicsCompressorNode（任意）
  - AudioDestination
```

### 8-4. ファイル構成

```text
volume-control/
├── manifest.json
├── service_worker.js
├── offscreen.html
├── offscreen.js
├── popup/
│   ├── popup.html
│   ├── popup.css
│   └── popup.js
└── icons/
    ├── icon16.png
    ├── icon48.png
    └── icon128.png
```

## 9. 技術要件

### 9-1. manifest

* Manifest Version 3 を使用する
* action popup を利用する
* 必要権限のみを宣言する

### 9-2. 想定権限

* `tabCapture`
* `offscreen`
* `activeTab`

※ 実装詳細によっては追加調整の可能性あり

### 9-3. 音声制御

* Web Audio API を使用する
* `GainNode` により音量倍率を調整する
* 100% = `1.0`
* 150% = `1.5`
* 600% = `6.0`

### 9-4. 音割れ軽減

* 初期実装では `GainNode` を中心に実装
* 必要に応じて `DynamicsCompressorNode` を後段に追加可能な構造とする
* 「音割れを完全に防ぐ」ことは保証対象外とする

---

## 10. 処理フロー

### 10-1. popup を開いたとき

1. アクティブタブ情報を取得
2. service worker に現在タブの状態を問い合わせ
3. 現在音量を popup に表示

### 10-2. 初回音量制御開始時

1. ユーザーが拡張アイコンをクリック
2. popup が開く
3. service worker が現在タブ用の stream ID を取得
4. offscreen document が存在しなければ生成
5. offscreen document で MediaStream を取得
6. AudioContext を構築
7. GainNode に現在音量を反映

### 10-3. スライダー操作時

1. popup で値変更
2. service worker 経由で offscreen にメッセージ送信
3. offscreen 側の GainNode を更新

### 10-4. Reset時

1. 音量を `1.0` に設定
2. UI 表示を `100%` に更新

---

## 11. エラーハンドリング

### 11-1. 対象タブで音声が取得できない場合

* popup に簡潔なエラー表示を行う
* 例: `このタブでは音量制御を開始できません`

### 11-2. offscreen 初期化失敗

* 再試行導線を用意する
* 必要なら「Reload」相当の軽い復旧ボタンは将来追加可能

---

## 12. 制約事項

* 音量の保存機能は実装しない（セッション内のみ有効）
* ドメイン単位保存やルールベース保存は実装しない
* イコライザ機能は実装しない
* タブ一覧表示は実装しない
* 複数タブ一括制御は実装しない
* 設定画面は作らない
* Chrome 以外のブラウザ対応は考慮しない

## 16. 最終方針

この拡張は、**「多機能な音量拡張」ではなく、タブの音量調整をシンプルUIで快適に行うための拡張** とする。

目的は以下の2点に絞る。

1. UIを見やすくする
2. タブごとの音量差を手軽に調整する
