# Simple Volume Control 要件定義書

## 1. 概要

既存の Chrome 拡張 **Volume Master** は便利だが、UI がごちゃごちゃしており、見た目・操作感の面で好みに合わないため、  
**「音量制御機能に特化し、UI を極力シンプルにした自作 Chrome 拡張機能」** を開発する。

本拡張は、単なるページ内 `<audio>` / `<video>` 要素の操作ではなく、  
**アクティブタブの音声そのものを対象にした音量制御** を行う方針とする。

これにより、サイト実装への依存を減らし、より安定した挙動と完成度を目指す。

---

## 2. 開発方針

- **完成度重視**
- **UI は最小限**
- **機能は音量制御に絞る**
- **動画ごとの個別音量保存を重視**
- 初期実装では **完全URL単位で保存**
- 技術構成は **B案（tabCapture + offscreen document）** を採用する

---

## 3. 基本仕様

| 項目 | 内容 |
|---|---|
| 拡張機能名 | Volume Control（仮） |
| 形式 | Chrome拡張機能（Manifest V3） |
| 対象ブラウザ | Google Chrome |
| 操作対象 | アクティブタブの音声 |
| 音量範囲 | 0% ～ 600% |
| デフォルト音量 | 100% |

| UI方針 | 小さく、ミニマル、視認性重視 |

## 4. この拡張で解決したいこと

### 4-1. 解決したい課題
- Volume Master の UI がごちゃごちゃしていて使いづらい
- サイト全体ではなく、**その動画だけ** 音量を上げ下げしたいことがある
- YouTube などでは動画ごとに音量感が異なるため、**動画単位で保存したい**
- 「この動画は毎回少し小さいから 150%」のような使い方をしたい

### 4-2. 保存仕様の考え方
- 保存は **完全URL単位** とする
- 例：
  - `https://www.youtube.com/watch?v=AAAA`
  - `https://www.youtube.com/watch?v=BBBB`
  は別の保存対象として扱う
- これにより、**動画ごとの個別最適化** を実現する
- サイト全体やドメイン単位の保存は初期実装では行わない

---

## 5. 機能要件

### 5-1. 音量スライダー
- 0% ～ 600% の範囲で音量を調整できる
- スライダー操作に応じて、リアルタイムで音量が変化する
- 現在値を `120%` のように数値表示する
- 小数は基本的に扱わず、表示は整数パーセントとする

### 5-2. リセットボタン
- クリックすると音量を **100%** に戻す
- スライダー表示も 100% に戻る
- 保存ON時は、保存値も 100% に更新される

### 5-3. 保存トグル
- デフォルトは **OFF**
- ON にすると、現在タブの完全URLに対して音量設定を保存する
- ON の状態で音量を変更した場合、保存値も即時更新する
- OFF に戻した場合、そのURLに紐づく保存値は削除する

### 5-4. 自動復元
- 保存済みURLを再度開いた場合、前回保存した音量を自動適用する
- 自動適用時は popup 上の表示も保存値に同期する
- 保存データが存在しない場合は 100% を適用する

### 5-5. 現在状態の表示
- popup を開いたとき、現在のタブに対する以下の情報を表示する
  - 現在音量（例: `150%`）
  - 保存状態（ON / OFF）
  - 現在のページが保存済みかどうか

---

## 6. 非機能要件

### 6-1. 使いやすさ
- 1画面で完結する
- 余計な機能を入れない
- 初見でも操作方法が分かる

### 6-2. 視認性
- ダークテーマ固定
- コントラストを明確にする
- 数値が見やすいことを優先する

### 6-3. パフォーマンス
- popup の表示は軽量であること
- 音量変更時の遅延を極力少なくする
- 通常利用でブラウザ操作を阻害しないこと

### 6-4. 保守性
- UI層と音声制御層を分離する
- popup 側に重い処理を持たせない
- 将来的な機能追加（プリセットやドメイン単位保存）に対応しやすい構成にする

## 8. 技術設計

### 8-1. 採用アーキテクチャ

**tabCapture + service worker + offscreen document + popup UI**

### 8-2. 採用理由

* popup は閉じると処理継続に向かないため、UI専用に分離したい
* 音量制御の本体は popup 外で動かしたい
* タブ音声全体を扱うことで、サイト依存を減らしたい
* 単純な content script より完成度を上げたい

### 8-3. 構成図

```text
popup (UI)
  ↓ メッセージ送信
service_worker
  ↓ stream ID 発行 / 状態管理
offscreen document
  ↓
Web Audio API
  - MediaStreamSource
  - GainNode
  - DynamicsCompressorNode（任意）
  - AudioDestination
```

### 8-4. ファイル構成

```text
volume-control/
├── manifest.json
├── service_worker.js
├── offscreen.html
├── offscreen.js
├── popup/
│   ├── popup.html
│   ├── popup.css
│   └── popup.js
└── icons/
    ├── icon16.png
    ├── icon48.png
    └── icon128.png
```

## 9. 技術要件

### 9-1. manifest

* Manifest Version 3 を使用する
* action popup を利用する
* 必要権限のみを宣言する

### 9-2. 想定権限

* `storage`
* `tabCapture`
* `offscreen`
* `activeTab`

※ 実装詳細によっては追加調整の可能性あり

### 9-3. 音声制御

* Web Audio API を使用する
* `GainNode` により音量倍率を調整する
* 100% = `1.0`
* 150% = `1.5`
* 600% = `6.0`

### 9-4. 音割れ軽減

* 初期実装では `GainNode` を中心に実装
* 必要に応じて `DynamicsCompressorNode` を後段に追加可能な構造とする
* 「音割れを完全に防ぐ」ことは保証対象外とする

### 9-5. データ保存

* `chrome.storage.local` を使用する
* 保存データは URL ごとに管理する
* popup / service worker / offscreen 間で状態同期する

---

## 10. データ仕様

### 10-1. 保存キー設計

保存は完全URL単位で行う。

例:

```json
{
  "volumes": {
    "https://www.youtube.com/watch?v=AAAA": {
      "volume": 1.5,
      "saved": true,
      "updatedAt": 1760000000000
    },
    "https://www.youtube.com/watch?v=BBBB": {
      "volume": 0.8,
      "saved": true,
      "updatedAt": 1760000001000
    }
  }
}
```

### 10-2. 保存値

* `volume`: 数値（0.0 ～ 6.0）
* `saved`: true / false
* `updatedAt`: 最終更新日時

### 10-3. 初期値

* 保存がなければ `1.0`
* Saveトグルは OFF 表示

## 11. 処理フロー

### 11-1. popup を開いたとき

1. アクティブタブ情報を取得
2. service worker に現在タブの状態を問い合わせ
3. 保存済みデータを確認
4. 現在音量と Save 状態を popup に表示

### 11-2. 初回音量制御開始時

1. ユーザーが拡張アイコンをクリック
2. popup が開く
3. service worker が現在タブ用の stream ID を取得
4. offscreen document が存在しなければ生成
5. offscreen document で MediaStream を取得
6. AudioContext を構築
7. GainNode に現在音量を反映
8. 必要なら保存値を適用

### 11-3. スライダー操作時

1. popup で値変更
2. service worker 経由で offscreen にメッセージ送信
3. offscreen 側の GainNode を更新
4. Save が ON なら storage を更新

### 11-4. Reset時

1. 音量を `1.0` に設定
2. UI 表示を `100%` に更新
3. Save ON 時は保存値も `1.0` に更新

### 11-5. Save ON時

1. 現在URLをキーとして保存
2. 現在音量を storage に記録
3. 以後の変更は自動更新

### 11-6. Save OFF時

1. 現在URLに紐づく保存値を削除
2. 以後はセッション内のみ有効とする

---

## 12. エラーハンドリング

### 12-1. 対象タブで音声が取得できない場合

* popup に簡潔なエラー表示を行う
* 例: `このタブでは音量制御を開始できません`

### 12-2. offscreen 初期化失敗

* 再試行導線を用意する
* 必要なら「Reload」相当の軽い復旧ボタンは将来追加可能

### 12-3. 保存データ破損時

* 当該URLデータのみ破棄し、100% にフォールバックする

---

## 13. 制約事項

* 初期実装では **完全URL単位保存のみ**
* ドメイン単位保存やルールベース保存は実装しない
* イコライザ機能は実装しない
* タブ一覧表示は実装しない
* 複数タブ一括制御は実装しない
* 設定画面は作らない
* Chrome 以外のブラウザ対応は考慮しない

## 17. 最終方針

この拡張は、**「多機能な音量拡張」ではなく、動画単位の音量調整をシンプルUIで快適に行うための拡張** とする。

目的は以下の3点に絞る。

1. UIを見やすくする
2. 動画ごとの音量差を吸収する
3. その動画専用の音量を保存して再利用できるようにする
